## 📌 创建 Personal Access Token（如需要）

如果 `git push` 时提示认证失败：

1. 访问：https://github.com/settings/tokens
2. 点击 **Generate new token** → **Generate new token (classic)**
3. Note：填写 `Vercel Deploy`
4. Expiration：选择 **90 days** 或 **No expiration**
5. 勾选权限：✅ **repo**（完整的仓库控制权限）
6. 点击 **Generate token**
7. **复制 Token**（只显示一次，请妥善保存！）

然后重新执行：
```bash
git push -u origin main
```

密码处粘贴刚才的 Token。

## ❓ 常见问题

### Q1: git push 时提示 "support for password authentication was removed"

**解决**：使用 Personal Access Token（见上方步骤）

### Q2: Vercel 部署失败

**解决**：
- 查看 Vercel Dashboard → 项目 → Deployments → **View Logs**
- 检查 `package.json` 中的 `build` 命令是否正确
- 确认 `dist` 文件夹中有 `index.html`

### Q3: 域名绑定后无法访问

**解决**：
- 检查 DNS 记录是否正确添加
- 等待 DNS 生效（最多 48 小时，通常几分钟）
- 查看 Vercel Domains 页面的验证状态

### Q4: 如何回滚到之前的版本？

**解决**：
- Vercel Dashboard → 项目 → Deployments
- 找到之前的部署
- 点击 **...** → **Promote to Production**
