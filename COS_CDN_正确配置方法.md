# COS CDN 加速配置指南（解决下载问题）

## 问题原因

COS 的静态网站功能 (`cos-website` 域名) 不能直接配置为 CDN 加速域名，会导致 "InvalidParameter.CdnHostIsCosDefaultAccess" 错误。

---

## 正确的配置方法（推荐）

### 方法一：在 COS 控制台开启默认 CDN 加速域名（最简单）

这是腾讯云官方推荐的方式，无需手动配置 CDN。

#### 操作步骤

1. **打开 COS 控制台**
   ```
   https://console.cloud.tencent.com/cos
   ```

2. **进入存储桶 `kunyuan-1257542161`**

3. **进入「域名与传输管理」**

4. **找到「默认 CDN 加速域名」**

   开启开关，系统会自动生成一个加速域名，格式类似：
   ```
   kunyuan-1257542161.file.myqcloud.com
   ```

5. **复制加速域名地址**

6. **访问新域名**

   等待 2-3 分钟后，直接访问这个新域名即可正常显示网站！

---

### 方法二：绑定自定义域名（如果有备案域名）

如果您购买了 `gdkunyuan.cn` 等已备案域名，可以绑定为 CDN 加速域名。

#### 前提条件
- 域名已完成实名认证和备案
- 备案审核通过（7-20 天）

#### 操作步骤

1. **在域名注册商添加 CNAME 解析**

   | 记录类型 | 主机记录 | 记录值 |
   |---------|---------|--------|
   | CNAME | www | `kunyuan-1257542161.file.myqcloud.com` |

2. **在 COS 控制台绑定自定义域名**

   - 存储桶 → 域名与传输管理 → **自定义域名**
   - 添加域名 `www.gdkunyuan.cn`
   - 选择 **CDN 加速**
   - 点击「提交」

3. **配置 HTTPS（可选）**

   - 绑定成功后，申请免费 SSL 证书
   - 在「HTTPS 配置」中开启 HTTPS

4. **访问**

   ```
   https://www.gdkunyuan.cn
   ```

---

## 为什么会下载页面？

COS 静态网站功能的 `Content-Type` 设置问题导致浏览器把 HTML 当成下载文件。

**解决方法：**
- 使用 COS 的「默认 CDN 加速域名」或「自定义 CDN 加速域名」
- CDN 会自动处理 Content-Type，无需手动配置

## 常见问题

### Q: 为什么不推荐用 `cos-website` 域名？
A: 该域名主要用于静态网站托管，CDN 不支持直接加速。建议用 `file` 域名或自定义域名。

### Q: 默认 CDN 加速域名要收费吗？
A: 有免费流量额度（每月 50GB），企业官网通常用不完。

### Q: 自定义域名一定要备案吗？
A: 是的，中国大陆访问的域名必须备案。

## 总结

| 方案 | 优点 | 缺点 | 适合人群 |
|------|------|------|---------|
| **默认 CDN 加速域名** | 开启即用，无需备案 | 域名较长，不够专业 | 快速测试 |
| **自定义域名 + CDN** | 专业大气，品牌感强 | 需要购买域名 + 备案 | 正式运营 |

**建议：** 先用「默认 CDN 加速域名」解决下载问题，域名备案通过后再绑定自定义域名。
