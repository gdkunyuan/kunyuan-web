# 最简单的 CDN 配置方案（解决下载问题）

## 为什么网页会下载？

COS 的静态网站功能 (`cos-website` 域名) 存在 Content-Type 问题，浏览器会把 HTML 当成文件下载。

---

## 最简单的解决方案：通过 CDN 控制台添加加速域名

您目前的情况：
- 已有 COS 存储桶：`kunyuan-1257542161`
- 已有文件：网站已上传到 COS

### 操作步骤（3 分钟）

#### 第一步：进入 CDN 控制台

打开 👉 **https://console.cloud.tencent.com/cdn**

#### 第二步：添加域名

点击 **「添加域名」**，填写如下：

| 配置项 | 填写值 |
|--------|--------|
| **加速域名** | `kunyuan-1257542161.cos.ap-guangzhou.myqcloud.com` |
| **业务类型** | 静态加速 |
| **加速区域** | 中国大陆 |
| **所属项目** | 默认项目 |

#### 第三步：配置源站（关键！）

展开 **「源站配置」** 部分：

| 配置项 | 填写值 |
|--------|--------|
| **源站类型** | 选择 **COS 源** |
| **回源协议** | HTTP |
| **源站地址** | 从下拉列表中选择 `kunyuan-1257542161`（广州） |
| **私有存储桶访问** | **开启** |

> 如果提示需要授权，按提示前往 COS 控制台对 CDN 服务进行授权（一键授权）

#### 第四步：完成添加

确认所有配置正确，点击 **「提交」**

---

## CDN 生效后

等待 5-10 分钟，访问新地址：

```
http://kunyuan-1257542161.cos.ap-guangzhou.myqcloud.com
```

**CDN 会自动处理 Content-Type，不会再下载页面了！**

---

## 为什么这个方案有效？

| 对比项 | cos-website 域名 | CDN 加速域名 |
|--------|------------------|-------------|
| Content-Type | 可能不正确，导致下载 | CDN 自动处理，正确识别 |
| 访问速度 | 普通访问 | 全国 CDN 加速，更快 |
| 是否推荐 | ❌ 不推荐 | ✅ 推荐 |

---

## 如果添加失败

如果提示 "域名已被占用" 或其他错误，说明之前已经添加过了。直接进入 **「域名管理」** 查看已有域名状态。

---

## 备选方案：临时解决下载问题

如果 CDN 配置遇到问题，可以先临时修复：

### 手动设置 index.html 的 Content-Type

1. 进入 COS 控制台 → 存储桶 `kunyuan-1257542161`
2. 进入 **「文件列表」**
3. 找到 `index.html` → 右侧 **「更多」** → **「设置元数据」**
4. 添加或修改 `Content-Type`：
   ```
   text/html; charset=utf-8
   ```
5. 点击 **「保存」**

这样可以临时解决下载问题，但**建议还是配置 CDN**，访问体验会更好。
